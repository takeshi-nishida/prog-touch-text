# 演習問題「計算で図形を描く」

[計算で図形を描く](02-pattern-drawing) の範囲の演習問題です。

## 授業でひな型プログラムを配布された方向けの説明
問題番号とひな型プログラムの収録されたフォルダ名が対応しています。 
そのフォルダにある sketch.js を編集することを想定しています。 プログラム内にもコメントでヒントが書かれている場合や、プログラムの一部が既に書かれている場合があります。

## 問題2-1 確認問題

![](https://www2.kobe-u.ac.jp/~tnishida/programming/images/circles.png)

まずは小手調べです。内側に青い円が5つ、外側に赤い円が5つです。 円を描くには `ellipse(中心のx座標, 中心のy座標, 幅, 高さ);` を使います。 色を赤にするには `stroke(255, 0, 0);`、青くするには `stroke(0, 0, 255);` とします。 内側の円から先に描くと後から描いた円に塗りつぶされてしまいますので、 塗りつぶしをしないようにする `noFill();` を事前にしておくか、外側の円から順に描くようにするかする必要があります。

ProgTouch の通常モードと挑戦モードで頭と心の準備をしてから実際にプログラムを書いてみましょう。

<ProgTouchEmbed>
<Step thoughts="円は ellipse(中心のx座標, 中心のy座標, 幅, 高さ); で描ける。まずは1個描いてみよう。">

function setup(){
  createCanvas(100, 100);
  // @A

</Step>

<Step undo={1} thoughts=""  />
<Step thoughts="円の中心は同じ、大きさは変わるから変数を使おう" at="A">
```javascript
  for(let i = 0; i < 10; i++){
    // @B
  }
```
</Step>

<Step thoughts="あれ？１つしかない！大きい円に小さい円が塗りつぶされてしまった！" at="B">
```javascript
    let size = (i + 1) * 10;
    // @C
    ellipse(50, 50, size, size);
```
</Step>

<Step thoughts="塗りつぶし無しにしたら無事10個に　塗り分けは条件分岐を使おう" at="C">
```javascript
    noFill();
```
</Step>
<Step thoughts="i を使って条件分岐を書いたら完成！" at="C">
```javascript
    if(i < 5){
      stroke(0, 0, 255); // 青
    } else {
      stroke(255, 0, 0); // 赤
    }
```
</Step>
</ProgTouchEmbed>


## 問題2-2 チェッカー

![](https://www2.kobe-u.ac.jp/~tnishida/programming/images/checker.png)

二重ループと条件分岐を使う問題です。まずは格子模様だけを描いてみましょう。i行j列の色は「i+jが偶数か？」で分岐するとうまく塗分けられます。

## 問題2-3 国旗

国旗の中には繰り返しパターンがあるものが多いです。図形の個数が何かしらの意味を持っている場合もあります。 国旗を再現するプログラムを描くことでそういったことも知ることができて一石二鳥ですね！ （余談ですが、企業ロゴマークには繰り返しはあまり見られません。なぜでしょうね。）
